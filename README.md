### 💡 深度思考：指针与防御性编程
在学习指针的第三个作用时，我理解了“信号与数据分离”的设计思想：
- **Return 0/1**: 作为防御性信号，告知调用者函数执行的状态。
- **Pointer (*res)**: 真正作为“远程机械臂”修改内存中的内容物。
- **逻辑转换**: 巧妙利用 `if(!flag)` 将工程上的“0(成功)”转换为机器的“真(执行)”。

、[深度理解 void]*
void* 是“万能接口”，负责接收任何人的地址。
char* 是“通用手术刀”，负责把任何数据拆解成 1 字节。
len 是“任务清单”，告诉我们要拆解多少个字节。
原理：只要把组成数据的每一个字节都换了，数据本体也就换了。这就是“化整为零”的思维。

[二级指针终极总结]
想改变量的值（改钱）：传一级指针，函数内用 *p。
想改指针的指向（换人）：传二级指针，函数内用 *pp。
面试必考题：为什么 scanf("%d", &a) 要传地址？
进阶回答：因为 scanf 需要修改我 main 里的变量。同理，如果我要一个函数修改我 main 里的指针，我就必须传指针的地址，也就是二级指针。
